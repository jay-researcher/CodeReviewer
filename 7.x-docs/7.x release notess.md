# CodeReviewer 7.x Release Notes

> 本文档记录 CodeReviewer 7.x 每次发布的功能、行为变化、配置变化、验证结果和部署状态。
>
> 维护规则：后续每次 7.x 版本更新，都必须同步更新本文档和《CodeReviewer User Mannual》。未明确执行生产部署时，Release Notes 必须标记为“仅本地”，不得暗示已经部署到 `192.168.3.78`。

## 7.0.4 — 2026-07-16

发布状态：已发布到本地环境 `127.0.0.1:8765`；未部署到 `192.168.3.78`。

### 新增与改善

- 同一个 Jira Issue 同时涉及前端和后端 MR 时，报告按照 `responsible + project type` 分离问题清单。
- 支持从 `config.yml` 读取 `type: frontend` 和 `type: backend`。
- 支持两种配置结构：
  - CodeReviewer/Git Tools 扁平 repository 配置；
  - JiraReviewer 嵌套项目配置，repository 可继承上层项目的 `type` 和 `responsible`。
- 即使前端、后端属于同一负责人，也会生成相互独立的问题清单。
- 拆分报告文件名增加类型，例如：
  - `ECHNL-8888_frontend_has-issue-high.md`
  - `ECHNL-8888_backend_has-issue-critical.md`
- 报告 Basic Information 增加 `Project Type`，便于确认分类结果。
- 当前 `config.yml` 已补充 DPS、DPS build、iTrade Client、WVAdmin 等项目的前后端类型。

### 使用说明

需要聚合同一 Jira Issue 的所有关联 MR，并按前后端拆分报告时，使用 Jira Review：

```powershell
python review.py --jira ECHNL-5657
```

`--mr-url` 仍只审核明确指定的单个 MR，不会自动扩展为整个 Jira Issue 的前后端 MR 集合。

### 验证

- 自动化测试：101 项通过。
- 当前配置解析：识别 51 个 repository，并能正确区分 frontend/backend。
- 本地版本接口：`GET http://127.0.0.1:8765/api/version` 返回 `7.0.4`。
- Git commit：`1c9913a fix: split Jira reports by frontend and backend`。

## 7.0.3 — 2026-07-16

发布状态：已部署到本地环境和 `192.168.3.78:8765`。

### 新增与改善

- 默认 LLM Provider 调整为 `codex-cli`，通过 CPA Responses API 执行审核。
- 禁止 CPA/Codex 失败后自动 fallback 到 CC Switch，避免额外等待以及不同 Provider 产生不一致报告。
- CC Switch 仅在显式配置 `LLM_PROVIDER=cc-switch` 时使用。
- DPS 项目发生 Codex 300 秒硬超时后，不再重复执行无效的同类请求。
- 完成 RHEL9 生产环境从 6.23.0 到 7.0.3 的升级，同时保留生产用户验证文件和既有数据。

### 验证

- 生产健康检查返回版本 `7.0.3`。
- CPA/Codex 连通性检查通过。
- 部署前 RHEL9 测试通过。

## 7.0.2 — 2026-07-16

### 新增与改善

- Windows 进程可读取用户级 `OPENAI_API_KEY`，改善旧 PowerShell/VS Code 会话看不到新环境变量的问题。
- 优化 Finding Handling 布局：提交按钮固定在问题标题右上角，普通处理不再保留无意义的空白右栏。
- “不是阻碍，另报 Jira”调整为更清晰的左右双栏布局，并提供 Jira Follow-up 卡片。
- 修正 `web-sv-build/dps` 等 DPS 根项目识别。
- 收敛 DPS Codex 超时后的重试和 Provider fallback 行为。

## 7.0.1 — 2026-07-16

### 新增与改善

- Windows 本地 CodeReviewer 默认通过 CLIProxyAPI `127.0.0.1:8318/v1` 调用 Codex Responses API。
- CPA API Key 由外部 `OPENAI_API_KEY` 注入，不写入 Git 仓库。
- 补充 Codex 300 秒超时诊断说明。
- 补充 Jira Issue、Sprint、Filter Review 的 MR 多来源发现顺序、范围过滤、分支类型路由及关闭 Sprint 后的诊断方法。

## 7.0.0 — 2026-07-15

### 新增与改善

- 建立 Developer、Auditor/Leader、Manager 三角色权限模型。
- 增加首批 Developer 账户及 responsible 映射。
- 增加 Issues Review History、Review Run、Finding lineage 和状态生命周期。
- 实现 `Handling → Re-scan → Problems → Handling` 闭环。
- Critical/High 的 Fixed 必须经过后续 Re-scan 验证。
- Developer 提交 Not an issue 后必须由 Auditor/Manager 审批。
- 增加 Manager Exception、Manual Pass 及审计记录。
- 增加 Report/Issue Discussion 和 AI Follow-up。
- 增加 Pending Jira 草稿及 ADF 可视化编辑、预览、版本控制和附件能力。
- ADF Expand 支持表格、有序列表、无序列表和截图。
- 增加 SQLite 工作流数据库、历史迁移及未来数据库适配边界。
- 密码存储升级为 PBKDF2-SHA256 Hash。
- 增加 GIT_VERSION Release Gate、锁定仓库检查和独立 LLM 上下文预算。
- 完成 Review Communication、Finding Handling、Issue Reviews、Tabs、卡片和安全间距等 Web 体验优化。
